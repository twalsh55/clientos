====================================
HUMAN APPROVAL WORKFLOW

The agent must NEVER autonomously publish public posts without explicit approval from the founder.
All suggested posts must first be sent to the founder via Telegram for review and approval.

APPROVAL FLOW:

1. Generate proposed post
2. Assign unique suggestion ID number
3. Send suggestion to founder via Telegram
4. Wait for approval command
5. Only publish after approval is received

APPROVAL COMMAND FORMAT:

- `/approve <id>`

Example:

- `/approve 42`

Only after receiving a valid approval command may the agent publish the associated post.

REJECTION / REVISION COMMANDS:

- `/reject <id>`
- `/revise <id>`

If rejected:

- do not publish
- optionally generate improved alternatives

If revised:

- regenerate the post using feedback/context
- send updated version for approval again

====================================
TELEGRAM MESSAGE FORMAT

Each Telegram approval request must include:

- suggestion ID
- target platform
- proposed post
- posting rationale
- expected learning objective
- confidence score
- estimated spam risk
- estimated engagement quality

Example:

Suggestion #42
Platform: Reddit
Goal:
Validate operational pain around invoice reconciliation workflows.

Proposed Post:
"How are finance teams handling invoice reconciliation today? Curious how many people still rely on spreadsheets/manual checks."

Expected Signal:

- workflow pain validation
- existing tool discovery
- willingness-to-pay indicators

Confidence: 8/10
Spam Risk: Low

Approve with:
`/approve 42`

Reject with:
`/reject 42`

Request revision with:
`/revise 42`
